## Goal: Full Reproducibility

Every analysis, preprocessing step, and model training must be **fully reproducible**:
- Running same code with same data produces identical results (to numerical precision)
- Any team member can re-run pipeline and get same outputs
- No "randomness" in outputs (all randomness is seeded)
- Results don't depend on machine, order of execution, or package versions

## 1. Random Seed Management

### 1.1 Seed Values (Fixed)

All random seeds must be set to **fixed constants** (NOT generated randomly):

```python
# In every script that uses randomness:
import numpy as np
import torch
import random

RANDOM_SEED = 42  # Fixed constant, same for all runs

def set_seeds(seed=RANDOM_SEED):
    """Set all random seeds for reproducibility."""
    random.seed(seed)
    np.random.seed(seed)
    torch.manual_seed(seed)
    if torch.cuda.is_available():
        torch.cuda.manual_seed(seed)
        torch.cuda.manual_seed_all(seed)
    
    # Ensure deterministic behavior (slower, but reproducible)
    torch.backends.cudnn.deterministic = True
    torch.backends.cudnn.benchmark = False

# Call at start of every script
set_seeds(RANDOM_SEED)
```

### 1.2 Where Seeds Apply

**Scripts requiring seeds:**
- `preprocess_lstm.py`: For any random sampling (stratified splits)
- `preprocess_xgb.py`: For any random sampling
- `train_lstm.py`: LSTM weight initialization, dropout, optimization
- `train_xgb.py`: XGB tree generation, feature sampling
- `evaluate.py`: Any bootstrapping or Monte Carlo sampling
- `predict.py`: Deterministic model loading/inference path

**Check before implementation:**
```python
# Verify seeds are set at script start
assert np.random.get_state()[1][0] == [42 or RANDOM_SEED]  # numpy
assert torch.initial_seed() == [42 or RANDOM_SEED]  # torch
```

# Verify no data leakage (must match every run)

## 8. What Should Be Reproducible?

**Should match perfectly:**
- Preprocessed data arrays (same values, same order)
- Data splits (train/val/test indices identical)
- Train loss curves (same values per epoch)
- Model weights (initialized same way)
- Test metrics (RMSE, CRPS, same values)

**May vary slightly (acceptable):**
- GPU training on different hardware (floating point precision)
- Small differences <1e-6 in metrics
- Order of operations in parallel code

**Must NOT vary:**
- Data splits (train/val/test boundaries exact)
- Quantile calibration (PIT, coverage exact)
- Per-region metrics (reproducible fairness check)

## 9. Common Reproducibility Failures

| Issue | Cause | Fix |
|-------|-------|-----|
| Different random numbers each run | Seeds not set | Call `set_seeds()` at script start |
| Different preprocessing each time | Recomputing instead of loading | Save preprocessing, load cached version |
| Library version mismatch | Ranges in pyproject.toml | Pin exact versions with `==` |
| Different train loss | Optimizer state not seeded | Set seed before optimizer creation |
| Different data splits | Shuffling timestamps | Use explicit date ranges, never shuffle |
| Floating point noise in metrics | Expected on GPU | Acceptable if <1e-6 difference |
